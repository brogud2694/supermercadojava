Novedades del controlador JDBC

En las secciones siguientes se explican las características nuevas de la versión 2.0 del controlador JDBC de Microsoft SQL Server.

Novedades de la versión 2.0 del controlador JDBC

La versión 2.0 del controlador JDBC de Microsoft SQL Server incluye varias características nuevas y mejoras.

Nombre del controlador

El nombre del controlador JDBC ya no contiene el identificador de versión de SQL Server. En consecuencia, la versión 2.0 del controlador JDBC de Microsoft SQL Server es la versión más reciente.

Compatibilidad con JDBC 4.0

La versión 2.0 del controlador JDBC proporciona ahora compatibilidad con la API 4.0 de JDBC, que incluye las siguientes características nuevas.

  • Compatibilidad con la API del conjunto de caracteres nacionales
    La versión 2.0 del controlador JDBC incluye ahora nuevos métodos de la API de conversión de conjuntos de caracteres nacionales. Esta compatibilidad incluye nuevos métodos establecedor, captador y actualizador para los tipos de JDBC NCHAR, NVARCHAR, LONGNVARCHAR y NCLOB. Para obtener más información, consulte Compatibilidad con juego de caracteres nacionales.

  • Compatibilidad con SQLXML
    La versión 2.0 del controlador JDBC proporciona ahora compatibilidad con los tipos de datos SQLXML. Para obtener más información acerca de cómo usar SQLXML, vea Compatibilidad con datos XML.

  • Los métodos SQLServerConnection createBlob, createClob y createNClob
    La versión 2.0 del controlador JDBC mejora ahora la clase SQLServerConnection al permitir la creación de los objetos BLOB, CLOB y NCLOB. Para obtener más información, vea los métodos createBlob, createClob y createNClob.

  • Interfaz java.sql.Wrapper
    La versión 2.0 del controlador JDBC es compatible con la interfaz java.sql.Wrapper. Esta interfaz proporciona un mecanismo para obtener accesso a las extensiones para la API de JDBC que son específicas del controlador JDBC de Microsoft SQL Server mediante una interfaz de proxy.
    La interfaz java.sql.Wrapper define dos métodos: isWrapperFor y unwrap. El método isWrapperFor comprueba si el objeto de entrada especificado implementa esta interfaz. El método unwrap devuelve un objeto que implementa esta interfaz para permitir el acceso a los métodos específicos del controlador JDBC de Microsoft SQL Server.

  • Otros métodos nuevos de la API de JDBC 4.0
    La versión 2.0 del controlador JDBC proporciona compatibilidad con los siguientes métodos de JDBC 4.0:
    • SQLServerConnection: isValid, getClientInfo y setClientInfo.
    • SQlServerDatabaseMetadata: autoCommitFailureClosesAllResultSets, getClientInfoProperties, getFunctionColumns, getFunctions, getRowIdLifetime, getSchemas y supportsStoredFunctionsUsingCallSyntax.
    • SQLServerStatement: método isClosed.
    • SQLServerCallableStatement: los métodos sobrecargados setAsciiStream, setBinaryStream, setCharacterStream, setBlob, setClob y el método getCharacterStream.
    • SQLServerPreparedStatement: los métodos sobrecargados setAsciiStream, setBinaryStream, setCharacterStream, setBlob y setClob.
    • SQLServerResultSet: isClosed, getHoldability, y los métodos sobrecargados updateAsciiStream, updateBinaryStream, updateBlob, updateCharacterStream y updateClob.

JDBC 3.0 Jar y JDBC 4.0 Jar

La versión 2.0 del controlador JDBC proporciona dos archivos de la biblioteca de clases, sqljdbc.jar y sqljdbc4.jar, que hay que usar dependiendo de su configuración preferida de Java Runtime Environment (JRE).

La biblioteca de clases sqljdbc.jar proporciona compatibilidad con JDBC 3.0 y necesita Java Runtime Environment (JRE) de la versión 5.0. La biblioteca de clases sqljdbc4.jar proporciona compatibilidad con JDBC 4.0 y necesita Java Runtime Environment (JRE) de la versión 6.0 o posterior. sqljdbc4.jar incluye todas las características de sqljdbc.jar, así como los nuevos métodos de JDBC 4.0. Para obtener más información acerca del archivo JAR que hay que elegir, vea Requisitos del sistema para el controlador JDBC.

Importante: La versión 2.0 del controlador JDBC no es compatible con JRE 1.4.

Propiedad de conexión sendStringParametersAsUnicode

Para obtener un rendimiento óptimo de los tipos de datos CHAR, VARCHAR y LONGVARCHAR de JDBC, una aplicación que use la versión 2.0 del controlador JDBC debería configurar la propiedad de conexión sendStringParametersAsUnicode en "false" y usar métodos de caracteres que no sean nacionales. Para obtener más información sobre la propiedad de conexión sendStringParametersAsUnicode, vea Establecer las propiedades de conexión.

Propiedad de conexión loginTimeout

Desde la versión 2.0 del controlador JDBC, el valor predeterminado de la propiedad de conexión loginTimeout es de 15 segundos. En la versión 1.2, el valor predeterminado era 0. Para obtener más información acerca de la propiedad de conexión loginTimeout, vea Establecer las propiedades de conexión.

Seguimiento mejorado del funcionamiento del controlador

Esta versión del controlador JDBC mejora el seguimiento del funcionamiento del controlador al registrar los puntos de entrada y salida de los métodos públicos, y al proporcionar una mejor distinción entre los niveles de seguimiento. Para obtener más información, vea Hacer un seguimiento del funcionamiento del controlador.

Comportamiento predeterminado del almacenamiento en búfer adaptable

Desde la versión 2.0 del controlador JDBC, el comportamiento predeterminado del almacenamiento en búfer de respuesta es "adaptive.". Esto significa que para obtener el comportamiento de almacenamiento en búfer adaptable la aplicación no tendrá que solicitarlo explícitamente. En la versión 1.2, el modo del almacenamiento en búfer era "full" de manera predeterminada y la aplicación tenía que solicitar el modo de almacenamiento en búfer adaptable explícitamente. Si desea mantener en su aplicación el comportamiento predeterminado de la versión 1.2, debe configurar la propiedad de conexión responseBufferring en "full". Para obtener más información acerca del almacenamiento en búfer adaptable, vea Usar el almacenamiento en búfer adaptable.

Compatibilidad con intercalación

En la versión 2.0 del controlador JDBC, el controlador admite todas las intercalaciones compatibles con SQL Server 2000 y SQL Server 2005, y las nuevas intercalaciones o nuevas versiones de los nombres de intercalación de Windows que se incluyeron en SQL Server 2008. Para obtener más información, vea Características internacionales del controlador JDBC.
